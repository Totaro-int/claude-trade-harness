# claude-trade-harness

**LLM-in-the-loop 트레이딩 하네스 + BYO(Bring-Your-Own) 전략.** 증권사 OpenAPI 문서 URL과 키를 넣으면 Claude가 브로커 어댑터를 자동 생성·검증해 연결하고, 자연어 전략 문서를 두뇌(Claude)에 물려 페이퍼 트레이딩으로 돌리는 로컬 골격입니다.

---

> **면책 고지**
>
> 이 프로젝트는 교육·연구 목적의 오픈소스입니다. 매매 판단과 그로 인한 모든 손실은 사용자 본인의 책임이며, 어떠한 투자 수익도 보장하지 않습니다.
> 기본 동작 모드는 **페이퍼 트레이딩(가상 체결)** 이며, 실거래 전환은 명시적 잠금 해제가 필요합니다. 실거래 전환은 권장하지 않습니다.

---

## 이 프로젝트의 정체

**진짜 제품은 "하네스(harness)"입니다.** 배우고 포크할 가치가 있는 것은 다음입니다.

- **브로커 무관 어댑터 자동생성** — 증권사 OpenAPI 문서만으로 TypeScript 어댑터를 생성하고, 정적 검사·컴파일·읽기전용 연결 테스트로 검증합니다.
- **코드로 강제되는 가드레일** — 돈이 움직이는 경로는 AI가 아닌 결정론적 TypeScript로만 구성됩니다(포지션·주문 한도, 일일 손실 한도 등).
- **관측성** — 모든 판단·체결·스냅샷이 기록되고, 토스 스타일 로컬 대시보드에서 실시간으로 보입니다.
- **claude-in-the-loop 골격** — 시세·포지션·전략을 프롬프트로 묶어 Claude의 판단을 받아오는 사이클 엔진.

**내장 전략은 돈 버는 시스템이 아닙니다.** 기본 제공되는 전략(대형주 지표 + LLM 판단)은 **참고용 베이스라인이며, 거래비용을 못 이기는 것으로 알려진 접근**입니다. 이건 출발점일 뿐이고, 전략을 `strategy/*.md`로 직접 교체(BYO)하는 것이 이 도구의 전제입니다.

**정직한 점수판은 벤치마크 대비 수익률입니다.** 대시보드는 동일가중 buy-and-hold 대비 초과수익을 보여줍니다. 이 값은 **대부분 음수일 것이고, 그게 정상입니다.** 절대 수익률이 아니라 이 벤치마크 대비 값으로 전략을 평가하세요.

> ### ⚠️ 주문 경로 미검증 경고
>
> **생성된 어댑터의 `submitOrder`는 한 번도 실증된 적이 없습니다.** 연결 검증 게이트는 읽기전용 3종(인증 · 시세 · 잔고)만 통과시킵니다. 즉, 주문 제출 코드는 단 한 번도 실행되지 않은 채로 어댑터가 "연결됨" 상태가 됩니다.
> 라이브로 전환하기 전(v1은 어차피 라이브 주문을 차단합니다) **생성된 `adapters/<broker>/adapter.ts`를 반드시 사람이 직접 검토하세요.** 자동 생성 코드를 검증 없이 신뢰하지 마세요.

## 무엇인가요?

claude-trade-harness는 세 가지 단계로 동작합니다.

1. **온보딩** — 증권사 OpenAPI 문서 URL과 API 키를 입력하면, Claude가 해당 API를 분석해 TypeScript 어댑터 코드를 자동 생성합니다. 어댑터는 정적 검사·컴파일·읽기전용 연결 테스트를 모두 통과해야 활성화됩니다. (단, 위 경고대로 주문 경로는 검증 대상이 아닙니다.)
2. **전략 설정** — 자연어로 작성된 투자 전략 문서(Markdown)를 업로드합니다. 내장 베이스라인을 그대로 쓰지 말고 본인 전략으로 교체하는 것을 권장합니다.
3. **운영** — 장중 30분 간격으로 Claude가 현재 시세·지표와 전략을 조합해 매매 판단을 내리고, 코드 가드레일을 통과한 경우에만 가상 체결합니다. 토스 스타일 로컬 대시보드에서 실시간으로 확인할 수 있습니다.

## 시작하기

**요구사항:** Node.js 20+, [Claude Code CLI](https://claude.ai/code) 로그인 상태(구독 사용)

```bash
git clone <repo> && cd claude-trade-harness
npm install
npm start
```

브라우저에서 `http://localhost:3000` 온보딩 페이지로 접속합니다.

## Claude Code 플러그인 (선택)

터미널에서 `npm start`를 직접 치는 대신, **슬래시 커맨드**로 온보딩·운영·상태조회·백테스트를 제어할 수 있는 얇은 런처 플러그인을 함께 제공합니다 (`plugins/trade-harness`).

```text
/plugin marketplace add Totaro-int/claude-trade-harness
/plugin install trade-harness@claude-trade-harness
```

| 커맨드 | 동작 |
|--------|------|
| `/trade-setup` | 첫 온보딩 — 의존성 설치 후 온보딩 서버 시작 |
| `/trade-start` | 운영 데몬 백그라운드 시작 + 대시보드 |
| `/trade-status` | 자산·일손익·벤치마크 대비 초과수익·포지션 요약 |
| `/trade-stop` | 데몬 정상 종료 |
| `/trade-backtest [막대수]` | 과거 캔들 백테스트 |

플러그인은 하네스 앱을 `$CLAUDE_TRADE_DIR` → 현재 디렉토리 → `~/open-trader` 순으로 자동 탐색합니다. 자세한 내용은 [`plugins/trade-harness/README.md`](./plugins/trade-harness/README.md).

> 플러그인은 **런처일 뿐**이며 매매 엔진은 이 저장소(앱)가 백그라운드 데몬으로 실행합니다.

## 동작 원리

### ① 온보딩

```
문서 URL + API 키 입력
    → Claude가 어댑터 코드 생성
    → 정적 검사 (import/process/fetch/eval 금지 키워드 탐지)
    → TypeScript 컴파일 통과
    → 읽기전용 연결 테스트 (인증 → 시세 → 잔고)
    → 통과 시 어댑터 활성화
```

### ② 운영

```
장중 30분 간격
    → Claude가 시세 + 포지션 + 전략 기반 판단
    → 코드 가드레일 7가지 통과 확인
    → 통과분만 페이퍼(가상) 체결
    → 대시보드 SSE 실시간 업데이트
```

### ③ 안전 설계

- **결정론적 경로**: 돈이 움직이는 코드 경로는 AI 생성 코드가 아닌 정적 TypeScript로만 구성됩니다.
- **기본값 = 안전**: 모든 에러, 예외, 예상치 못한 상황의 기본 동작은 **거래 안 함** 입니다.
- **라이브 미실행 (v1)**: v1은 3중 잠금을 모두 통과하더라도 **실제 라이브 주문을 실행하지 않습니다(페이퍼만).** 이건 의도된 안전장치입니다.
- **라이브 3중 잠금**: (향후 라이브 지원 시) 실거래 전환을 위해서는 config.json `mode: "live"` 설정 + 환경변수 `LIVE_CONFIRM=1` + API로 계좌번호 끝 4자리 확인, 세 가지가 모두 맞아야 합니다.
- **지표 필수 (`requireIndicators`, 기본 `true`)**: 어댑터가 `getCandles`를 구현하지 않아 지표 데이터가 없으면 그 사이클은 **매매하지 않고 건너뜁니다.** lastPrice만으로 하는 매매는 검증된 우위가 없고 수수료만 까먹는 노이즈 트레이딩이므로 기본 차단합니다. (정 필요하면 `config.json`에 `"requireIndicators": false`.)

## 보안 / 시크릿

- 시크릿(`BROKER_API_KEY` 등)은 `.env` 파일에만 저장합니다.
- `adapters/`, `strategy/`, `data/`, `.env`는 `.gitignore`에 등록되어 있습니다. 포크·공유 시 개인정보가 포함되지 않습니다.
- 생성된 어댑터는 로드 시마다 정적 검사를 재실행합니다(`import`, `process`, `fetch`, `eval` 등 금지).
- 어댑터의 외부 요청은 설정된 `baseUrl`(http/https 한정)로만 허용됩니다.

> **잔여 리스크 고지**: 어댑터가 정당하게 받은 시크릿(API 키 등)을 브로커 `baseUrl`로 전송하는 것은 설계상 필요한 동작이며, 정적 검사로 막을 수 없습니다. **라이브 모드 첫 실행 전, 생성된 `adapters/<broker>/adapter.ts` 파일을 사람이 직접 검토할 것을 강력히 권장합니다.**

## 가드레일 설정

`config.json`에서 아래 항목을 조정할 수 있습니다.

| 항목 | 기본값 | 설명 |
|------|--------|------|
| `maxPositionPct` | `20` | 종목당 최대 비중 (%) |
| `maxOrderPct` | `10` | 1회 주문 금액 상한 (%) |
| `maxOrdersPerCycle` | `3` | 사이클당 최대 주문 수 |
| `dailyLossLimitPct` | `3` | 일일 손실 한도 (%) — 도달 시 당일 거래 중단 |
| `maxOrdersPerDay` | `10` | 하루 최대 주문 수 |
| `reentryCooldownMin` | `60` | 매도 후 동일 종목 재매수 금지 시간 (분) |
| `maxTotalExposurePct` | `80` | 총 주식 노출 상한 (%) |
| `minHoldMin` | `0` | 이익 실현 매도의 최소 보유시간 (분) — 회전율 억제. **손절(평가손) 매도는 항상 면제.** `0`이면 비활성 |

그 외 동작 플래그(`config.json` 최상위):

| 항목 | 기본값 | 설명 |
|------|--------|------|
| `reflection` | `true` | 청산된 매매의 thesis 적중/실패를 회고로 누적해 다음 판단 프롬프트에 주입(과거 매매에서 학습) |
| `skepticGate` | `false` | BUY 결정마다 2차 '회의론자' 반박 검토를 1회 더 거침(돌이킬 수 없는 매수만 재검증). **매수마다 claude 1회 추가 호출** — 비용 발생, 백테스트엔 미적용 |
| `holidays` | (없음) | 기본 KRX 휴장일에 **추가**할 휴장일 `["YYYY-MM-DD", …]`(임시공휴일·선거일 등). 기본 휴장일 표는 `src/core/market-calendar.ts`에 있으며 매년 공식 캘린더로 검증·갱신 필요 |
| `alertWebhook` | (없음) | 운영 알림을 보낼 webhook URL(Slack/Discord/generic, `{ text }` JSON POST). 미지정 시 콘솔·macOS 로컬 알림만 |
| `retentionDays` | `90` | `decisions`·`snapshots` 로그 보존 일수 — 장마감마다 이보다 오래된 행 정리(거래·KV는 보존) |

## 무인 운영 (몇 주 자동 운영)

장기 무인 페이퍼 운영을 위한 안전장치:

- **거래일 캘린더** — 주말·KRX 휴장일에는 매매하지 않습니다(묵은 시세 매매 방지). 추가 휴장일은 `holidays`로 보강.
- **`GET /health`** — liveness 엔드포인트. 건강하면 `200`, 연속 사이클 실패(3회 이상)나 사이클 멈춤 의심 시 `503`. 마지막 성공/실패 사이클·연속실패·가동시간을 JSON으로 반환합니다.
- **원격 알림** — claude 인증 만료, 연속 사이클 실패, 치명적 오류(미처리 예외) 시 `alertWebhook`으로 알립니다.
- **크래시 안전** — 미처리 예외/거부 시 브로커 상태를 저장하고 알림 후 종료 → 외부 supervisor가 재시작.

외부 감시(crontab 예시 — 5분마다 `/health` 폴링해 실패 시 알림):

```bash
*/5 * * * * curl -fsS http://127.0.0.1:3000/health >/dev/null || \
  curl -s -X POST "$ALERT_WEBHOOK" -H 'content-type: application/json' -d '{"text":"open-trader /health DOWN"}'
```

자동 재시작 supervisor(예: pm2):

```bash
pm2 start "npm start" --name open-trader --time
pm2 save && pm2 startup   # 부팅 시 자동 기동
```

## 전략 평가 (백테스트)

전략을 실거래/페이퍼에 붙이기 전에 **위험 없이 과거 데이터로 평가**하려면 백테스트를 사용하세요. 과거 캔들을 실제와 동일한 PaperBroker(수수료·세금·스프레드)·가드레일·프롬프트로 리플레이하고 성과 리포트를 출력합니다.

```bash
npm run backtest            # 기본 60 막대
npm run backtest -- --bars 120
```

평가 기준은 절대 수익률이 아니라 **동일가중 buy-and-hold 벤치마크 대비 초과수익(%p)** 입니다 — 리포트에서 가장 크게 표시되는 "정직한 점수판"이며, 대부분의 경우 음수일 것이고 그게 정상입니다.

> 주의 ①: LLM 두뇌(`claude -p`)를 막대당 1회 호출하므로 막대 수만큼 호출/비용/시간이 듭니다 (장기 구간은 `--bars`를 작게).
> 주의 ②: 일봉 백테스트는 매수를 **당일 종가**에 체결하는 표준 단순화를 씁니다 — 절대 수익률은 다소 낙관적일 수 있으나, 벤치마크도 같은 기준이라 **초과수익 비교는 공정**합니다.

### 전략/설정 변형 비교 (eval)

전략 문서나 가드레일 설정의 **어느 변형이 더 나은지**를 같은 캔들·같은 두뇌로 백테스트해 초과수익으로 줄세웁니다. `strategy/eval/` 폴더에 변형 전략문서(`*.md`)를 넣으면 각 파일이 하나의 변형이 되고, 현재 전략(`base`)과 함께 비교됩니다.

```bash
npm run eval                # base + strategy/eval/*.md 변형들을 비교
npm run eval -- --bars 120
```

> 변형마다 막대당 claude를 1회 호출하므로 비용은 (변형 수 × 막대 수)에 비례합니다. LLM은 비결정적이라 단일 실행엔 노이즈가 크니, **절대값보다 변형 간 상대 순위**로 해석하고 막대 수를 늘려 완화하세요.

# 내 전략
