# claude-trade-harness

**AI 가치투자 분석 하네스.** 증권사 OpenAPI 문서 URL과 키를 넣으면 Claude가 브로커 어댑터를 자동 생성·검증해 연결하고, **내 실제 보유 종목을 받아 가치투자 4대가(버핏·멍거·돤융핑·리루) 관점으로 평가**해 "들고 갈지·더 살지·줄일지·팔지"를 진단 리포트로 뽑아주는 로컬 골격입니다.

> [ai-berkshire](https://github.com/xbtlin/ai-berkshire)의 4대가 리서치 방법론을 이식했습니다.

---

> **면책 고지**
>
> 이 프로젝트는 교육·연구 목적의 오픈소스입니다. 산출되는 분석은 **투자 자문이 아니며**, 모든 투자 판단과 손실은 사용자 본인의 책임입니다. 어떠한 수익도 보장하지 않습니다.
> 이 도구는 **주문을 실행하지 않습니다.** 토스 OpenAPI에서 보유·시세를 **읽기만** 하고, 결과는 마크다운 리포트로 출력합니다. 돈이 자동으로 움직이는 경로가 없습니다.

---

## 이 프로젝트의 정체

**진짜 제품은 "분석 하네스"입니다.** 배우고 포크할 가치가 있는 것은 다음입니다.

- **브로커 무관 어댑터 자동생성** — 증권사 OpenAPI 문서만으로 TypeScript 어댑터를 생성하고, 정적 검사·컴파일·읽기전용 연결 테스트로 검증합니다.
- **4대가 멀티렌즈 분석** — 보유 종목마다 4명의 가치투자 대가 관점으로 Claude가 병렬 웹리서치하고, team-lead가 종합 판정을 내립니다.
- **반편향(anti-bias) 강제** — 정반양면(매수·매도 논거 동반), 정보충실도 등급(A/B/C), 반증조건, "추측으로 프레임 채우지 않기"를 프롬프트로 강제합니다.
- **검산 게이트** — 시총 같은 핵심 수치는 LLM의 "검산했다"는 말을 믿지 않고, 코드(`verifyMarketCap`)가 `시가 × 주식수`를 직접 교차검증해 환각을 🚨로 잡습니다.

**이건 돈 버는 시스템이 아닙니다.** 가치투자 분석을 **구조화·재현 가능**하게 만드는 도구일 뿐, 종목 추천·수익을 보장하지 않습니다. 분석의 확실성은 자료량이 아니라 비즈니스 모델 자체에서 옵니다.

> ### ⚠️ 펀더멘털은 웹리서치에 의존합니다
>
> 토스 OpenAPI는 시세·보유·캔들·호가만 줄 뿐 **재무제표·해자·경영진 같은 펀더멘털을 주지 않습니다.** 4대가 분석의 펀더멘털은 **Claude의 웹 검색**으로 채웁니다. 즉 분석 품질은 공개 웹 정보의 양·정확도에 좌우되며, 한 번의 LLM 실행은 비결정적이고 노이즈가 있습니다. 핵심 수치는 검산 게이트로 일부 거르지만, **모든 출처를 직접 검증한 결과로 신뢰하지 마세요.**

## 무엇인가요?

claude-trade-harness는 두 단계로 동작합니다.

1. **온보딩(`npm run onboard`)** — 증권사 OpenAPI 문서 URL과 API 키를 입력하면, Claude가 해당 API를 분석해 TypeScript 어댑터 코드를 자동 생성합니다. 어댑터는 정적 검사·컴파일·읽기전용 연결 테스트(인증·시세·잔고)를 모두 통과해야 활성화됩니다.
2. **분석(`npm run review`)** — 토스에서 실보유 종목·현금을 받아, 종목마다 4대가 렌즈가 웹리서치로 분석하고, 종합 판정과 포트폴리오 진단을 `reports/portfolio-latest.md`로 출력합니다.

---

## 시작하기

**요구사항:** Node.js 20+, [Claude Code CLI](https://claude.ai/code) 로그인 상태(구독 사용)

```bash
git clone <repo> && cd claude-trade-harness
npm install
npm run onboard       # http://localhost:3000 에서 어댑터 온보딩
npm run review        # 온보딩 후 — 보유종목 분석 → reports/portfolio-latest.md
```

## 동작 원리

### ① 온보딩

```
문서 URL + API 키 입력
    → Claude가 어댑터 코드 생성
    → 정적 검사 (import/process/fetch/eval 금지 키워드 탐지)
    → TypeScript 컴파일 통과
    → 읽기전용 연결 테스트 (인증 → 시세 → 잔고)
    → 통과 시 어댑터 활성화
```

### ② 분석 파이프라인 (`npm run review`)

```
토스 getBalance()  → 실보유 N종목 + 현금
  └ 종목마다 4대가 렌즈 병렬 (웹검색 허용 claude -p)
       business · financial · industry · risk
  └ team-lead 종합 → 준입/조건부/회색 + 목표가
       + add/hold/trim/exit + 확신등급(A/B/C) + 반증조건
  └ 검산 게이트 → financial 렌즈 수치를 verifyMarketCap으로 교차검증
  └ 포트폴리오 집계 → 집중도·기회비용·"가장 먼저 할 일"
  └ reports/portfolio-latest.md 출력
```

### ③ 4대가 렌즈

| 렌즈 | 마스터 | 분석축 |
|------|--------|--------|
| business | 돤융핑(段永平) | 비즈니스 모델·해자·차별화·가격결정력 |
| financial | 버핏(巴菲特) | 재무·수익성·현금흐름·안전마진·검산 |
| industry | 멍거(芒格) | 산업 격국·경쟁·밸류체인 |
| risk | 리루(李录) | 리스크·경영진·장기 확실성 |

### ④ 안전 설계

- **주문 경로 없음**: 이 도구는 읽기 전용입니다. `submitOrder`를 호출하지 않으며, 자동 체결·라이브 거래 기능이 없습니다.
- **검산 강제**: 시총·밸류 수치는 코드가 결정론적으로 교차검증합니다(LLM 신뢰 안 함).
- **결측 안전**: 일부 렌즈가 실패해도 나머지로 종합하며, 데이터 부족은 "데이터 부족"으로 정직하게 표기합니다.

## 비용 주의

`npm run review` 1회는 **종목당 claude를 약 5회**(4 렌즈 + 1 종합) 호출하며, 렌즈는 웹검색을 사용합니다. 보유 종목이 많으면 호출 수·시간·비용이 그만큼 늘어납니다(종목 1개 기준 수 분 소요). LLM은 비결정적이라 같은 종목도 실행마다 결과가 달라질 수 있습니다.

## 보안 / 시크릿

- 시크릿(`BROKER_API_KEY` 등)은 `.env` 파일에만 저장합니다.
- `adapters/`, `data/`, `.env`는 `.gitignore`에 등록되어 있습니다. 포크·공유 시 개인정보가 포함되지 않습니다.
- 생성된 어댑터는 로드 시마다 정적 검사를 재실행합니다(`import`, `process`, `fetch`, `eval` 등 금지).
- 어댑터의 외부 요청은 설정된 `baseUrl`(http/https 한정)로만 허용됩니다.

> **잔여 리스크 고지**: 어댑터가 정당하게 받은 시크릿(API 키 등)을 브로커 `baseUrl`로 전송하는 것은 설계상 필요한 동작이며, 정적 검사로 막을 수 없습니다. **생성된 `adapters/<broker>/adapter.ts` 파일을 사람이 직접 검토할 것을 권장합니다.**

## mock 모드로 데모 실행

브로커 연결 없이 온보딩/엔트리포인트를 확인하려면 `config.json`에 `{ "brokerId": "mock" }`를 둡니다. (mock 어댑터는 빈 보유를 반환하므로 실제 4대가 분석은 실보유가 있는 토스 연결에서 확인하세요.)

## 범위 밖

- **주문 실행·자동매매** — 이 도구는 분석만 합니다(설계상 주문 경로 없음).

- 결정론적 재무 데이터 소스(DART 등) 연동 — 현재 펀더멘털은 웹리서치 의존(후속 과제).
